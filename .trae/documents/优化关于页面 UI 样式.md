## 目标
- 向主流、简洁、卡片化的移动端风格靠拢（微信/抖音等小程序常见样式）。
- 保持现有技术栈（uni-app + UnoCSS + wot-design-uni），不引入新依赖。
- 信息层级更清晰：简介区 → 项目说明 → 功能特色 → 更多信息。

## 视觉规范
- 统一使用白底卡片：`rounded-3 bg-white p-4 shadow-sm`，页面背景使用浅灰：`bg-gray-100`。
- 统一标题样式：`text-4 font-semibold text-gray-800`；说明文字：`text-3.5 text-gray-600`；辅助信息：`text-3 text-gray-500`。
- 间距规范：区块之间 `mx-3 mb-3`，区块内元素 `gap-2~3`，按钮/可点击区 `active:opacity-80`。
- 轻量分隔：适当使用 `border-t border-gray-200` 或区块间距替代重分割线。

## 结构优化
- 顶部“应用介绍”做成信息卡：名称、版本、描述，保留“初版”标识为小标签（chip风格）。
- “项目说明”区：
  - 标题 + 说明文字 + 仓库信息卡（左名称，右复制链接按钮）。
  - 链接复制后 toast 提示（沿用现有 `uni.setClipboardData`）。
- “功能特色”区：
  - 保留列表，但每项采用更宽松的行高与更细的图标风格（继续用 ✓ 文字符号，避免新增组件依赖）。
- “更多信息”区：
  - 保持 `wd-cell-group`，统一 `title-width`，优化标签文案与层级。

## 组件与依赖约束
- 保留并内嵌现有 `demo-block` 包裹（当前项目仅本页使用，避免删除造成不确定影响）。
- 不新增 wot 组件类型（仓库检索未发现如 `wd-card`、`wd-divider` 的使用），以 UnoCSS 辅助完成风格统一。

## 交互与可用性
- 链接复制按钮添加按下态：`active:opacity-80`，容器支持点击范围更大（右侧按钮为主）。
- 保持现有模态与 toast 交互，不变更触发逻辑。

## 实施步骤
1. 统一区块卡片化：为三处区块容器补充 `shadow-sm`、标题样式与间距规范。
2. 优化“项目说明”文案与布局：展示仓库名与复制按钮，同步辅助说明文字。
3. 保留“功能特色”列表，调整每行的字号与色彩，使得层级更清晰。
4. 优化“更多信息”中的 cell 文案与一致性（仅样式与文案，不改交互）。
5. 全局检查字号、颜色与间距，确保与顶部介绍区保持一致。

## 验收标准
- 页面各区块均呈现白底圆角卡片，阴影统一且不突兀。
- 标题、说明、辅助文字的字号与颜色一致，层级清晰。
- 仓库复制按钮交互顺畅，toast 提示符合预期。
- 不新增依赖、不影响既有功能事件。

## 变更范围
- 仅改动 `src/pages/about/index.vue` 的模板与样式类名，复用现有逻辑函数。
